# パチスロ来店イベント分析アプリ - デザイン計画書

## 概要

パチスロユーザー向けの来店イベント情報集約アプリ。地図上で「アツい店舗」をビジュアル的に表示し、ユーザーを優良店舗へ誘導する。

## 画面構成（モバイルポートレート 9:16）

### 1. 地図画面（ホーム）
- **主要コンテンツ**: フルスクリーン地図表示
- **機能**:
  - 店舗ピン表示（アツさに応じた色・サイズ変化）
  - 現在地ボタン
  - 検索バー（上部）
  - フィルターボタン
- **レイアウト**:
  - 上部: 検索バー + フィルターアイコン
  - 中央: 地図（全画面）
  - 下部: タブバー

### 2. 店舗詳細画面（モーダル/プッシュ）
- **主要コンテンツ**: 店舗情報カード
- **機能**:
  - 店舗名・住所
  - 営業時間・台数
  - イベント情報リスト（日付、演者名）
  - 情報ソース表示
  - 外部リンクボタン
  - 有料掲載バッジ（該当店舗のみ）
- **レイアウト**:
  - ヘッダー: 店舗名 + アツさインジケーター
  - 本文: スクロール可能な情報カード
  - フッター: 外部リンクボタン

### 3. お気に入り画面
- **主要コンテンツ**: お気に入り演者・エリアリスト
- **機能**:
  - 登録演者一覧
  - よく行くエリア一覧
  - 好きな機種一覧
- **レイアウト**:
  - セグメントコントロール（演者/エリア/機種）
  - リスト表示

### 4. 設定画面
- **主要コンテンツ**: アプリ設定
- **機能**:
  - 通知設定（ON/OFF、時間帯）
  - 位置情報設定
  - アカウント情報
  - プライバシーポリシー
- **レイアウト**:
  - グループ化されたセクション
  - トグルスイッチ

## キーユーザーフロー

### フロー1: 店舗発見
1. アプリ起動 → 地図画面表示
2. 地図をパン/ズーム → 周辺店舗のピン確認
3. アツいピン（赤・大きい）をタップ
4. 店舗詳細画面表示
5. イベント情報確認 → 外部リンクで詳細確認

### フロー2: 演者検索
1. 検索バーをタップ
2. 演者名を入力
3. 該当イベントのある店舗を地図上にハイライト
4. 店舗タップ → 詳細確認

### フロー3: お気に入り登録（登録ユーザー）
1. 店舗詳細画面で演者名をタップ
2. 「お気に入りに追加」確認
3. お気に入り画面で確認可能

## デザインコンセプト

### パチスロ特有の派手さを表現
- **ダークテーマベース**: ネオンカラーが映える黒背景
- **グラデーション**: アツさを表現するグラデーション効果
- **高コントラスト**: 視認性の高いデザイン
- **メタリック質感**: ゴールド・シルバーのアクセント

### アニメーション
- ピンタップ時: スケールアップ + パルス効果
- 画面遷移: スライドイン
- アツいピン: 微細なパルスアニメーション

## 技術的考慮事項

### 地図実装
- `react-native-maps`を使用
- Expo Goでテスト時はAPIキー不要
- 本番デプロイ時はGoogle Maps APIキーが必要

### パフォーマンス
- 大量ピン表示時のクラスタリング検討
- 地図操作時の60fps維持

### データ
- MVPではモックデータを使用
- 将来的にバックエンドAPIと連携
