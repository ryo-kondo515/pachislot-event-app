# スクレイピング分析メモ

## 1. drillermaguro.com

### サイト構造
- トップページ: https://drillermaguro.com/
- 個別取材結果ページ: https://drillermaguro.com/interview/{slug}/

### 取得可能な情報
1. **トップページ（一覧）**
   - 店舗名（例：アビバ湘南台、プレゴ池袋南口）
   - エリア（例：神奈川、東京、宮城）
   - イベント日付（例：1月25日(日)）
   - イベントタイプ（例：ジャンドリ、マグロ、あがり、極上）
   - 投稿日（例：2026.01.29）

2. **詳細ページ**
   - 店舗名
   - イベント日付
   - イベントタイプ
   - 機種情報（台番、機種名、ゲーム数、差枚）
   - S級獲得台数（例：508台(20)）
   - 総流獲量データ

### スクレイピング方針
- **店舗情報**: トップページの一覧から店舗名とエリアを抽出
- **イベント情報**: 各取材結果リンクから日付、イベントタイプを抽出
- **アツさレベル**: イベントタイプに基づいて判定
  - マグロ: 5（超アツ）
  - ジャンドリ: 4（アツ）
  - 極上: 4（アツ）
  - あがり: 3（やや熱）
  - 海鮮ドン: 3（やや熱）

### 課題
- 店舗の住所・緯度経度情報が直接記載されていない
  → Google Maps APIまたはジオコーディングサービスで住所から緯度経度を取得する必要がある
- 演者情報が明示的に記載されていない
  → イベントタイプから推測するか、他のソースと組み合わせる
